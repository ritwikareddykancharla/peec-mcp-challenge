# Demo Script — Workflow 1

A 3-minute walkthrough you can record for your hackathon submission.

## Setup shot
- Claude Desktop open, "Linear — Citation Gap Briefs" project loaded.
- Show the Peec MCP server connected (green dot in connectors).
- Show the system prompt is set.

## Step 1 — Find the gaps (60s)

Type:

```
/find-gaps
```

**Expected output:** A ranked table of 8–15 domains where competitors get cited by AI engines but Linear doesn't. Top rows likely include G2, Capterra, Reddit, Hacker News, Zapier blog, Atlassian docs.

**Talking point:** "These are the exact domains feeding the AI when buyers ask about project management tools — and Linear isn't in the conversation on any of them."

## Step 2 — Generate a brief (90s)

Type:

```
/brief 1
```

**Expected output:** A full content brief with target prompts, competitor analysis (3 cited URLs scraped and summarized), recommended H1/H2 outline, internal linking plan, distribution recommendation.

**Talking point:** "In one prompt I went from 'we have a gap' to 'here's the article to write, who it's competing against, and where it should live.' This is what a content strategist would spend an afternoon on."

## Step 3 — Weekly automation (30s)

Type:

```
/weekly-gap-report
```

**Expected output:** A markdown report with top 5 new gaps this week, 3 quick wins, 2 long-term investments, and per-engine breakdown.

**Talking point:** "Run this every Monday morning. Paste into Notion. Hand to your content team. That's the workflow."
